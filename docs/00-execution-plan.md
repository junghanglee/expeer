# EXPEER 안정성 우선 개발 실행표

이 문서는 EXPEER 개발을 작은 단위로 쪼개서 하나씩 완료하기 위한 실행표다.

## 진행 원칙

- EXPEER는 AI Linker와 완전히 별도 프로젝트다.
- 경로는 `projects/expeer-lovable`만 사용한다.
- 안정성, 비수탁 원칙, 계좌 지급정지 방지, 증빙자료 확보를 우선한다.
- 서비스 구조가 바뀌는 결정은 사용자 확인 후 진행한다.
- MVP 핵심 기능을 먼저 처리하고, 고도화는 별도 제안 후 진행한다.

## B. 비수탁/온체인 안정화

### EX-003. 수수료 모델 확정

목표: 판매자/구매자 각각 1% 수수료를 코인으로 징수하는 구조를 확정한다.

상태: 사용자 결정 필요

결정 항목:

- 구매자 수수료를 별도 코인 납부로 받을지
- 정산 코인에서 자동 차감할지
- 판매자 lock 금액에서 통합 차감할지

추천:

- MVP는 정산 코인에서 판매자/구매자 수수료를 모두 자동 차감한다.
- 관리자에서는 `sellerFeeBps`, `buyerFeeBps`를 각각 설정한다.

완료 기준:

- 사용자 승인 후 컨트랙트/DB/관리자 설정 구조 반영

목표: 판매자가 안전하게 코인을 잠그고, 입금 확인 후 release하게 한다.

완료 기준:
