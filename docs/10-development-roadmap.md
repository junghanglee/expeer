# EXPEER 개발 로드맵

## 0. 제품 원칙

EXPEER는 비수탁 P2P 스테이블코인 거래 플랫폼이다.

핵심 원칙:

1. 비수탁
   - 플랫폼은 판매자/구매자의 코인과 현금을 직접 보관하지 않는다.
   - 법정화폐는 어떤 경우에도 EXPEER 계좌로 받지 않는다.
   - 코인은 온체인 스마트컨트랙트가 잠시 보관한다.

2. 비실명 거래 기조 유지
   - 사용자의 공개 거래 경험은 최대한 비실명/간편 구조를 유지한다.
   - 단, 불법자금 유입과 계좌 지급정지 위험을 줄이기 위한 최소 안전장치는 둔다.

3. 수수료 정책
   - 수수료는 코인으로만 징수한다.
   - 판매자와 구매자 모두에게 기본 1%씩 부과한다.
   - 수수료율은 관리자 화면에서 변경 가능해야 한다.

4. 블랙리스트/위험계좌 차단
   - haja2 같은 블랙리스트 제공 서비스와 연동 가능한 구조를 둔다.
   - 신고/위험 계좌는 AI 또는 룰 엔진이 자동 차단/검증한다.
   - 외부 제휴 전에는 내부 블랙리스트 테이블과 수동 등록 기능으로 시작한다.

5. 분쟁/사고 대응
   - EXPEER는 분쟁 판단자가 아니라 자료 제공자다.
   - 사용자와 판매자가 은행/경찰서에 제출할 수 있는 거래 자료 패키지를 쉽게 다운로드할 수 있어야 한다.

6. 쉬운 UI
   - 사용자는 복잡한 블록체인 용어를 몰라도 거래할 수 있어야 한다.
   - approve, lock, release 같은 온체인 단계는 쉬운 문구와 상태 안내로 감싼다.

## 1. 개발 운영 원칙

- 사용자는 개발 초보자이므로, 사용자가 직접 해야 하는 작업은 단계별로 자세히 설명한다.
- 서비스 구조가 바뀌는 주요 의사결정은 사용자 확인 후 진행한다.
- 중복 기능, 불필요한 기능 확장, 오버워킹을 피한다.
- 요구된 핵심 기능을 먼저 처리하고, 고도화 아이디어는 제안 후 승인받아 진행한다.

## 2. 권장 시스템 구조

```text
사용자 앱
  ├─ 회원가입/로그인
  ├─ 지갑 연결
  ├─ 은행계좌 등록 또는 거래용 계좌 정보 관리
  ├─ 판매 광고 등록
  ├─ 주문/채팅
  ├─ 입금완료/입금확인
  ├─ 에스크로 lock/release/refund
  └─ 증빙 패키지 다운로드

운영자 앱
  ├─ 수수료 설정
  ├─ 체인별 컨트랙트 주소 설정
  ├─ 지원 토큰 설정
  ├─ 주문/분쟁 조회
  ├─ 블랙리스트 계좌 관리
  ├─ 위험거래 모니터링
  ├─ 증빙 패키지 생성/다운로드
  └─ 인덱서 상태 확인

온체인
  ├─ ExpeerEscrowVaultV2
  ├─ 판매자 코인 lock
  ├─ 구매자 release
  ├─ 만료 refund
  ├─ arbiterResolve
  └─ 코인 수수료 자동 분배

오프체인
  ├─ Supabase DB
  ├─ 주문/채팅/계좌/증빙 저장
  ├─ 블랙리스트 검증
  ├─ 온체인 이벤트 인덱서
  └─ 위험 점수/자동 차단
```

### 003. 수수료 모델 재설계

목표: 판매자/구매자 각각 기본 1% 수수료를 코인으로 징수한다.

현재 이슈:

- 기존 문서에는 컨트랙트 수수료 기본값이 0.5%로 되어 있다.
- 현재 컨트랙트는 release 시 전체 amount에서 feeBps 1개만 차감한다.
- 사용자 요구사항은 판매자 1%, 구매자 1% 양쪽 수수료다.

결정 필요:

- 구매자 수수료를 어떻게 코인으로 받을지 구조 결정이 필요하다.

추천안:

A안. 판매자 lock 금액에서 총 2% 차감
- 장점: 구현이 가장 단순하다.
- 단점: 명목상 구매자 수수료를 판매자 코인에서 차감하는 구조라 정책 설명이 어색하다.

B안. 판매자 lock 시 판매자 수수료 1% 포함, 구매자에게는 수령액 기준 1% 차감
- 장점: 온체인에서 단순 처리 가능하다.
- 단점: 구매자가 별도로 낸다는 느낌은 약하다.

C안. 구매자도 별도 코인 지갑으로 수수료를 납부해야 release 가능
- 장점: 판매자/구매자 각각 징수 원칙이 가장 명확하다.
- 단점: 구매자는 원화 입금자인데 코인 수수료 지갑이 필요해 UX가 복잡해진다.

1차 추천:

- MVP는 B안으로 시작한다.
- UI에는 “거래 수수료는 정산 코인에서 자동 차감됩니다”라고 설명한다.
- 실제 정책/법무 검토 후 C안이 필요하면 별도 고도화로 분리한다.

완료 기준:

- 사용자 승인 후 컨트랙트/DB/관리자 수수료 구조를 확정한다.
